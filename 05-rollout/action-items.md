# Phase 5: Broad Rollout — Action Items

**Timeline:** Weeks 10–16  
**Customer:** Manufacturing Supplier (~250 devs, SOX/SOC 2/ITAR, Java/TS/Python/SQL)

## Champion Network Setup

- [ ] Recruit 4 champions for Wave 1 (1 per ~15 devs, backend + frontend coverage)
- [ ] Recruit 3 champions for Wave 2 (data eng + ML teams)
- [ ] Recruit 2 champions for Wave 3 / external (vendor liaison + integration lead)
- [ ] Train champions: advanced features, troubleshooting, feedback collection
- [ ] Set up champion Slack/Teams channel: `#copilot-champions`
- [ ] Define champion responsibilities: office hours, escalation path, monthly feedback report

## Wave 1: Backend + Frontend (Weeks 10–12)

- [ ] Add Wave 1 members to `SG-Copilot-Wave1` IdP group (60 seats activate)
- [ ] Deliver Copilot 101 training for new users (reuse pilot materials, incorporate lessons learned)
- [ ] Champions run team-specific onboarding sessions (Java, TypeScript focus)
- [ ] Monitor activation: >80% active users within 2 weeks
- [ ] Week 11 check-in: survey + usage dashboard review
- [ ] Address any adoption blockers surfaced by champions

## Wave 2: Data/ML + Remaining Internal (Weeks 12–14)

- [ ] Add Wave 2 members to `SG-Copilot-Wave2` IdP group (70 seats activate, 150 total)
- [ ] Deliver Python/data-specific training (Copilot for notebooks, data pipelines)
- [ ] Champions run onboarding for data/ML teams
- [ ] Monitor: total 150 internal devs active
- [ ] Run org-wide satisfaction survey at Week 13

## Wave 3: External Contractors (Weeks 14–16, conditional)

- [ ] **Pre-requisite:** Legal IP review complete and approved (from Phase 1 action item)
- [ ] If approved: draft contractor agreement addendum covering AI tool usage
- [ ] Get contractor companies to sign updated agreements
- [ ] Create `SG-Copilot-External` IdP group → map to external org GitHub Team
- [ ] Set **lower premium request limit** for external users (e.g., 50/month)
- [ ] Enable Copilot for external org — scoped to their permitted repositories only
- [ ] Verify content exclusions still block external users from internal-only repos
- [ ] Deliver training to external teams (condensed version)
- [ ] If NOT approved: document decision, set re-evaluation date (quarterly)

## Onboarding Guide Distribution

- [ ] Publish internal developer onboarding guide (install → authenticate → first use → tips)
- [ ] Add onboarding guide link to new-hire documentation
- [ ] Add Copilot setup to developer environment bootstrap scripts

## Deliverable

- [ ] 150 internal developers active on Copilot
- [ ] Champion network operational with regular cadence
- [ ] External contractor decision executed (enabled or documented deferral)
- [ ] Onboarding guide published and linked in standard docs
